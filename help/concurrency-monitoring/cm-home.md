# 同時実行性の監視の概要 {#intro}

同時実行性監視は、コンテンツプロバイダーおよび ID プロバイダー（MVPD およびプログラマー）が、複数のアプリケーション、デバイス、プラットフォームでの同時ビデオストリーミングに対する制限を定義し、適用できるようにするサービスです。 同時に視聴できるストリーム数を制御したいプログラマーや、MVPDがコンテンツパートナー全体に使用ポリシーを適用したい場合でも、同時実行性モニタリングは必要なツールを提供します。

## 同時実行性モニタリングとは {#what-is-cm}

同時実行性監視は、アクティブなビデオストリーミングセッションをリアルタイムで追跡および管理する一元化されたサービスです。 次の操作が可能です。

- **サブスクライバーごとの同時ストリームの制限** - ユーザーがアクセスできる同時ビデオストリーム数を制御します
- **デバイス制限の適用** - ストリーミングを特定のデバイスタイプまたは数量に制限します
- **場所ベースのポリシーの実装** – 地理的な場所に基づいてストリーミングを制限します
- **コンテンツ固有のルールの作成** - ライブコンテンツとVOD コンテンツに異なる制限を適用します
- **使用パターンの監視** - コンテンツがどのように消費されているかを把握します

## 仕組み {#how-it-works}

同時実行性の監視は、シンプルかつ強力な API を通じて動作します。

1. **セッションの初期化** - ユーザーがコンテンツの視聴を開始すると、アプリケーションによってセッションが作成されます
2. **ポリシー評価** – このサービスは、現在の使用状況に照らして定義済みのポリシーを評価します
3. **リアルタイム監視** - ハートビート呼び出しにより、セッションがアクティブな状態を維持し、コンプライアンスを監視
