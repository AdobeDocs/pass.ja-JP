# 同時実行性監視の基本を学ぶ {#getting-started-overview}

同時実行性モニタリングへようこそ。 このガイドは、基本事項を理解し、統合をすばやく開始および実行するのに役立ちます。

## 同時実行性の監視によって解決される問題 {#problem-solved}

現在のストリーミング環境では、購読者は複数のデバイスやアプリケーションをまたいでコンテンツにアクセスすることを想定しています。 ただし、これにより、コンテンツプロバイダーには次のような課題が生じます。

- **コンテンツライセンス契約** 多くの場合、同時ストリーム数は制限されます
- **帯域幅コスト** 過度の同時使用による増加
- **ユーザーエクスペリエンス** ストリームが多すぎると、パフォーマンスが低下します
- **歳入保護** アカウント共有の不正利用防止が求められる

同時実行性モニタリングは、これらの課題に対する一元的なソリューションを提供します。


## 同時実行性監視の仕組み

### コア機能

同時実行性監視は、次のような **セッション管理サービス** として動作します。

1. **アクティブなストリーミングセッションを追跡** すべてのアプリケーションにわたってリアルタイムで処理
2. 現在の使用パターンに照らして **ビジネスポリシーを評価**
3. ポリシーに違反した場合に **制限を適用**
4. **競合が発生した場合に解決オプションを提供**

## 様々な関係者にとってのメリット

### コンテンツプロバイダー（プログラマー）

- **コンテンツの権利の保護** ライセンス契約の遵守
- **過度な使用を防ぐことで、帯域幅コストを最適化**
- 制限に関する明確なフィードバックを使用して **ユーザーエクスペリエンスを向上**
- 詳細なレポートを通じて **使用状況に関するインサイトを取得**

### ID プロバイダー（MVPD）

- **購読者の契約の適用** すべてのコンテンツパートナーに適用
- **一元化されたポリシー管理** 複数のアプリケーションに対応
- 使用パターンの **リアルタイムモニタリング**
- **拡張性の高いアーキテクチャ** 数百万のセッションを処理

### エンドユーザー

- 使用制限の **明確な理解**
- **すべてのアプリケーションで一貫したエクスペリエンス**
- **制限に達した際に競合を解決するためのオプション**
- アクセスが制限される理由に関する **透過的なフィードバック**

## サービスの登録 {#service-registration}

同時実行性の監視を開始するには、以下の情報をアドビの [ サポートチーム ](mailto:tve-support@adobe.com) にお問い合わせください。

1. **会社名** 連絡先の詳細
2. **アプリケーション** 同時実行監視と統合する。 各アプリケーションに対して、以下を指定してください。
   1. アプリケーション名
   2. アプリケーションプラットフォーム
3. **統合パートナー** （他の関係者、プログラマー、MVPDの依頼で同時実行モニタリングを購読している場合）
