# ホームベース認証（HBA） {#home-based-authentication}

>[!IMPORTANT]
>
> このページのコンテンツは、情報提供のみを目的として提供されています。 このAPIを使用するには、Adobeの現在のライセンスが必要です。 無断使用は認められません。

Home-Based Authentication （HBA）は、有料テレビの加入者が、ホームネットワークに接続したときにMVPDの資格情報を入力することなく、TV EverywhereのコンテンツにオンラインでアクセスできるようにするTV Everywhere機能で、認証体験を大幅に向上させます。

Open Authentication Technology Committee （OATC）によると、

> 「在宅自動認証とは、MVPD/OVDがホームネットワークの特性（またはホームネットワーク上のデバイス間で自動的にアクセス可能な識別子）を使用して、そのホームネットワークに関連付けられている加入者アカウントを認証するプロセスであり、保護されたコンテンツにアクセスするためのTVE セッションを開始する際に利用者が手動で資格情報を入力する必要はありません。」

## HBAの利点 {#hba-benefits}

Home-Based Authentication （HBA）は、アクティブなケーブル サブスクリプションを使用して自宅の視聴者のログイン障壁を取り除く重要な機能です。 この障壁はTV Everywhere サービスにとって大きな課題であり、すべてのサインイン試みの約半数が失敗に終わっています。

HBAは、視聴者のエンゲージメントを大幅に向上させ、TV Everywhereのコンテンツにアクセスするためのシームレスで優れたユーザーエクスペリエンスを提供します。

## HBA サポート {#hba-support}

>[!IMPORTANT]
>
> HBA機能の活用に関心がある場合は、Adobe Pass Authenticationの営業担当者にお問い合わせください。詳しくは、特定のHBA フローがPremium Workflow パッケージに含まれていることをご確認ください。

HBAは、Adobe Pass Authenticationと統合された多くのMVPDでサポートされていますが、HBAのメリットを得るには、追加の手順を実行する必要がある場合があります。

**SAML MVPD**

SAML MVPDの場合、HBAはMVPD側でのみアクティブ化されます。

**OAuth2 MVPD**

OAuth2 MVPDの場合、[Adobe Pass TVE ダッシュボード ](https://experience.adobe.com/#/pass/authentication)を介して、[TVE ダッシュボード統合ユーザーガイド ](/help/authentication/user-guide-tve-dashboard/tve-dashboard-integrations.md#most-used-flows)の手順に従ってHBAのオンとオフを切り替えることができます。

### Adobe Pass 認証 {#hba-support-adobe-pass-authentication}

このセクションでは、HBA対応エクスペリエンスの概要と、Adobe Pass Authenticationが提供するサポートの詳細を説明し、次のような主な機能を紹介します。

* **HBA ID:**&#x200B;認証がHBAであるか非HBAであるかをプログラマに示す機能（MVPD サポートが必要）。
* **設定可能な認証TTL:** HBA認証と非HBA認証の異なる認証有効期間（TTL）値を設定できます（MVPD サポートが必要です）。

次の表に、HBAおよび通常（非HBA）認証フローにおけるユーザーエクスペリエンスの概要を示します。

| 認証タイプ | ユーザーエクスペリエンス |
|---------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| HBA | 利用者はMVPDを選択し、ホームネットワークに接続されると自動的に認証されます。 認証が期限切れになると、ユーザーはMVPDを再度選択する必要があり、その後、自動的に再認証されます。 |
| 標準（非HBA） | ユーザーはMVPDを選択し、ホームネットワークへの接続に関係なく、資格情報を入力するよう求められます。 認証が期限切れになると、ユーザーはMVPDを再度選択し、資格情報を再入力する必要があります。 |

>[!IMPORTANT]
>
> OAuth 2.0認証プロトコルを使用するMVPDのHBA フローでは、MVPDはビジネス要件によって定義されたTTLを含むリフレッシュトークンを発行し、AdobeはリフレッシュトークンのTTLを超えないTTLを含むHBA認証トークンを発行します。

## FAQ {#faqs}

1. SAML プロトコルとOAuth2 プロトコルに対するHBA実装の違いは何ですか？

   SAML用のホームベース認証（HBA）とOAuth2 プロトコルの分離は、これらのプロトコルが認証メカニズム、設定、実装の柔軟性の点で異なる動作を行うため存在します。 SAML MVPDの場合は、プログラマがHBAを有効にするための操作は必要ありません。一方、OAuth2 MVPDの場合は、[Adobe Pass TVE ダッシュボード ](https://experience.adobe.com/#/pass/authentication)を介してHBAのオンとオフを切り替えることができます。

1. HBAが有効になっている場合、ユーザーは初期認証時にユーザー名とパスワードを入力する必要がありますか？

   いいえ、ユーザー名とパスワードは必要ありません。

1. ペアレンタルコントロールを適用する方法

   Adobe Pass Authenticationでは、ペアレンタルコントロールの承認が必要なチャネルとの統合でHBAを無効にすることができます。 また、私たちはOATCと協力して、ペアレンタルコントロールを使用してHBA エクスペリエンスを設定する方法を推奨するUX ドキュメントに取り組んでいます。

1. HBAをサポートしているプロバイダーは、通常の認証と比較して、HBAに短いTTL ウィンドウを使用しますか？

   TTL設定は設定可能です。 誤った処理を防ぐために、MVPDとHBAの統合に対して短いTTLを設定することをお勧めします。
