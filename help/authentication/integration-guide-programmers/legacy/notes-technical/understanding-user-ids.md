# （レガシー） ユーザーIDについて {#understanding-user-ids}

>[!NOTE]
>
>このページのコンテンツは、情報提供のみを目的として提供されています。 このAPIを使用するには、Adobeの現在のライセンスが必要です。 無断使用は認められません。

>[!IMPORTANT]
>
> [製品のお知らせ](/help/authentication/product-announcements.md) ページに集計されている最新のAdobe Pass認証製品のお知らせと廃止予定について、常に情報を得てください。

概念的には、使用権限フローを開始する各ユーザーは、一意の単一のユーザーIDに関連付けられます。 ただし、エンタイトルメントフローの過程で、IDを取得するAPIに応じて、1つのユーザーIDをさまざまな方法で表示できます。

ショートメディアトークンの`sessionGUID`は、ユーザーIDの安全な形式で、`sendTrackingData()`呼び出しを介して利用できます。 現在のすべての統合では、これは時間とデバイスをまたいでユーザーの永続的なGUIDです。 GUIDのソースは、MVPDからの認証応答のUser IDで始まります。 ただし、一部のMVPDは将来、その考えを変え、一時的なGUIDの送信を開始する可能性があります。 プログラマーがAuthN応答のMVPD ソースユーザーIDが永続的であることを確認したい場合は、MVPDとの契約書でそれを整理する必要があります。

Adobe Pass Authentication APIでユーザーIDを表す様々な方法を次に示します。

| プロパティ | 目的 | ハッシュ化 | デジタル署名済み | 説明 |
| --- | --- | --- | --- | --- |
| sendTrackingData （） GUID プロパティ | トラッキング/分析 | はい | いいえ | - Adobeによってハッシュ化されたMVPD ユーザーID。 ユーザーIDは、MVPDのソースに遡ることができません。</br> </br> – この形式のIDはデジタル署名されていないため、不正防止のために安全ではありません。 分析には十分です。 </br> </br> – この形式のユーザーIDは、Adobe Pass AuthenticationがAuthN/AuthZ フローで生成するすべてのイベントでクライアント側で提供されます。 |
| Short Media TokenのsessionGUID プロパティ | 同時利用の不正追跡 | はい | はい |  – これはsendTrackingData （）を介したユーザーIDと同じですが、このIDはその整合性を保護するためにデジタル署名されており、不正行為の追跡に使用するのに十分です。</br> </br> – 検証ライブラリを使用した後、サーバーサイドで処理することを目的としており、ビデオストリームをクライアントに公開する前に不正行為パターンについて分析できます。  これらの作業を行うのは、プログラマの責任です。 |
| getMetadata （） userID プロパティ | アカウントリンク、MVPDによる不正調査 | いいえ | いいえ |  – このプロパティを使用すると、Adobeは実際のソース MVPD User IDをプログラマに公開できます。</br> </br> - Adobeの設定では、暗号化されているかどうかを設定できます（MVPDの環境設定によって異なります）。 暗号化されている場合は、Adobeに提供されたプログラマーの証明書の公開鍵で暗号化されるので、クライアントに明確に公開されません。</br> </br> – これにより、プログラマーはMVPDから実際のユーザーIDを取得できるので、これはアカウントのリンクや不正行為の調査に直接MVPDで使用できます。 |


**結論**

* 一般的に、MVPDは永続的な一意のID <u>を提供し、正常な認証</u>時にAdobeに渡します。 それは全てのネットワークで共通している。 例外は、チャンネルごとに異なるユーザーIDを提供するComcast MVPDです。

* MVPD ユーザーIDにはPIIが含まれておらず、アカウント番号でもありません。 PIIが送信されていないすべてのMVPDで検証されているため、暗号化された形式で公開する必要はありません。

ユーザーIDの使用方法は、次のユースケースによって異なります。

* トラッキングや分析に必要な場合は、最も実用的な場所は`sendTrackingData()`から取得することです。
* ストリームのリリース、不正行為、または運用データのためにサーバーサイドで必要な場合は、Media Token バリデーターから取得できます。
* アカウントのリンクや不正行為の深刻化に対応する必要がある場合は、Adobeの担当者にお問い合わせください。
