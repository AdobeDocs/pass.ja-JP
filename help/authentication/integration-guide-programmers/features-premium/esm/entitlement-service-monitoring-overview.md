# 使用権限サービスの監視の概要 {#entitlement-service-monitoring-overview}

>[!NOTE]
>
>このページのコンテンツは、情報提供のみを目的として提供されています。 このAPIを使用するには、Adobeの現在のライセンスが必要です。 無断使用は認められません。

## 概要 {#introduction}

TVEのサイトやアプリは24時間365日利用可能である必要があるため、お客様は問題をできるだけ迅速に検出して修正するために、insightをリアルタイムでエンタイトルメントイベントに利用する必要があります。 また、月間データを分析して、どのプラットフォームがトラフィックの大部分を提供しているか、どのプラットフォームの実装が悪く、コンバージョン率が低い可能性があるかを判断する必要もあります。

エンタイトルメントサービスモニタリング（ESM）は、プログラマーとMVPDに、認証イベントと認証イベントをリアルタイムで可視化するデータフィードを提供します。 データはAdobe Pass認証システムから収集され、RESTful APIを通じて提供されます。  顧客は、データを直接的に、または独自のカスタム構築された運用ダッシュボードの中から利用できます。

ESM システムの中核的な要素は、その指標と寸法です。 ESMは、ディメンション選択に従って、集計指標を含むレポートを生成します。 Adobe Pass イベントはPST タイムゾーンで記録されるため、ESM レポートはPST タイムゾーンでも使用できます。

ESM APIは一般には使用できません。  ご利用に関するご質問は、Adobeの担当者までお問い合わせください。

## プログラマー向けESM {#esm-for-programmers}

### プログラマは次の指標を監視できます。 {#programmers-monitor-metrics}


| *指標の名前* | *説明* |
|-------------------------|--------------------------|
| 試行の許可 | 開始された認証フローの数 |
| Authn-successful | クライアントが正常に取得した認証トークンの数 |
| オーサリング保留 | 正常に生成された認証トークンの数（クライアントが実際に取得したかどうかを無視します） |
| authn-failed | 外部システムを介して実行された失敗した認証の数。 |
| clientless-tokens | 正常に発行されたクライアントレス トークンの数 |
| clientless-failures | クライアントレス APIからトークンを受信するために失敗した試行の数 |
| authz-attempts | 認証の試行回数 |
| authz-successful | 成功した承認の数 |
| authz-failed | アプリケーションレベルでのMVPDによる拒否された承認の数 |
| authz-rejected | Adobe サービスプロバイダーによって悪意があると見なされ、DoS攻撃防止の結果として拒否された認証の試みの数 |
| authz-latency | MVPDのエンドポイントに費やした合計時間（ミリ秒） |
| media-tokens | 生成されたショートメディアトークンの数（再生リクエストの数と同義） |
| unique-accounts | 選択した時間間隔で使用権限（AuthN / AuthZ）アクションを実行したユニークユーザーの数。 （この指標は、1日の値が要求された場合にのみ表示されます。） </br> これは個々のデータセンターごとに計算されます。 「dc」ディメンションが要求されない場合、この指標は表示されません。 |
| unique-sessions | 選択した時間間隔でAdobe Pass Authentication サービスへの認証フロー呼び出しを実行した一意のセッションの数。 （この指標は、1日の値が要求された場合にのみ表示されます。） </br> これは個々のデータセンターごとに計算されます。 「dc」ディメンションが要求されない場合、この指標は表示されません。 |
| カウント | イベント指向レポートで使用される単純なカウンター |

## ユースケース {#use-cases}

ESM データは、次のユースケースで使用できます。

- **監視** - オペレーション チームまたは監視チームは、APIを毎分呼び出すダッシュボードまたはグラフを作成できます。 表示された情報を使用すると、表示された瞬間に（Adobe Pass認証またはMVPDで）問題を検出できます。

- **デバッグ/品質テスト** - データもプラットフォーム、デバイス、ブラウザー、OSごとに分割されているため、使用パターンを分析すると、特定の組み合わせ（OSXのSafariなど）に関する問題を特定できます。

- **Analytics** – 提供されたデータは、Adobe Analyticsまたは他の分析ツールを通じて収集されるクライアントサイドデータを補完または監査するために使用できます。
